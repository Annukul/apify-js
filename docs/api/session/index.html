Session

Class aggregating data for session. Sessions are used to store information such as cookies and can be used for generating fingerprints and proxy sessions. You can imagine each session as a specific user, with its own cookies, IP (via proxy) and potentially a unique browser fingerprint. Session internal state can be enriched with custom user data for example some authorization tokens and specific headers in general.

new exports.Session(options)

Session configuration.

ParamType
options
options.idString

Id of session used for generating fingerprints. It is used as proxy session name.

options.maxAgeSecsNumber

Number of seconds after which the session is considered as expired.

options.userDataObject

Object where custom user data can be stored. For example custom headers.

options.maxErrorScorenumber

Maximum number of marking session as blocked usage. If the errorScore reaches the maxErrorScore session is marked as block and it is thrown away.

options.errorScoreDecrementnumber

It is used for healing the session. For example: if your session is marked bad two times, but it is successful on the third attempt it's errorScore is decremented by this number.

options.createdAtDate

Date of creation.

options.expiredAtDate

Date of expiration.

options.usageCountNumber

Indicates how many times the session has been used.

options.errorCountNumber

Indicates how many times the session is marked bad.

options.maxSessionUsageCountNumber

Session should be used only a limited amount of times. This number indicates how many times the session is going to be used, before it is thrown away.

options.sessionPoolEventEmitter

SessionPool instance. Session will emit the sessionRetired event on this instance.

session.isBlocked() ⇒ boolean

indicates whether the session is blocked. Session is blocked once it reaches the maxErrorScore.

session.isExpired() ⇒ boolean

Indicates whether the session is expired. Session expiration is determined by the maxAgeSecs. Once the session is older than createdAt + maxAgeSecs the session is considered expired.

session.isMaxUsageCountReached() ⇒ boolean

Indicates whether the session is used maximum number of times. Session maximum usage count can be changed by maxSessionUsageCount parameter.

session.isUsable() ⇒ boolean

Indicates whether the session can be used for next requests. Session is usable when it is not expired, not blocked and the maximum usage count has not be reached.

session.markGood()

This method should be called after a successful session usage. It increases usageCount and potentially lowers the errorScore by the errorScoreDecrement.

session.getState() ⇒ Object

Gets session state for persistence in KeyValueStore.

Returns: Object - represents session internal state.

session.retire()

Marks session as blocked and emits event on the SessionPool This method should be used if the session usage was unsuccessful and you are sure that it is because of the session configuration and not any external matters. For example when server returns 403 status code. If the session does not work due to some external factors as server error such as 5XX you probably want to use markBad method.

session.markBad()

Increases usage and error count. Should be used when the session has been used unsuccessfully. For example because of timeouts.
